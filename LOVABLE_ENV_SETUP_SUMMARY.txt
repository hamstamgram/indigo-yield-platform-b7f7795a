================================================================================
SECURITY NOTES
================================================================================

1. NEVER commit real credentials to git
   - Add config/lovable-env-vars.json to .gitignore
   - Use secure storage for credential file

2. Always rotate credentials BEFORE production
   - Generate new keys in source systems
   - Don't reuse old credentials across environments
   - Document rotation date and reason

3. Mark secret variables in Lovable
   - SUPABASE_SERVICE_ROLE_KEY
   - POSTHOG_API_KEY
   - Prevents accidental exposure in logs

4. Implement regular rotation
   - Quarterly minimum (recommended)
   - Mandatory after security incidents
   - Document audit trail for compliance

5. Least privilege principle
   - Use anon key in browser (limited permissions)
   - Use service role key on server only (full permissions)
   - Never expose service role key to client

TROUBLESHOOTING
================================================================================

Issue: App won't load (401/403)
Solution: Verify VITE_SUPABASE_ANON_KEY is correct
  - Check no extra spaces when copied
  - Test key in Supabase dashboard first
  - Regenerate if needed

Issue: Errors not in Sentry
Solution: Verify Sentry DSN
  - Check format: https://[key]@o[org].ingest.sentry.io/[project]
  - Verify project is active
  - Check org ID is correct

Issue: Analytics missing from PostHog
Solution: Verify PostHog keys
  - Check VITE_POSTHOG_KEY matches project
  - Verify POSTHOG_API_KEY is correct
  - Check VITE_POSTHOG_HOST is reachable

Issue: Deployment fails with env var errors
Solution: Check configuration
  - No PLACEHOLDER_ values remain
  - No trailing spaces in values
  - JSON is valid: python3 -m json.tool config/lovable-env-vars.json
  - All 10 variables added to Lovable
