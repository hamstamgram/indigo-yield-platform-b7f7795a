# Indigo Yield QA Report — 2026-04-24

## Bugs Found & Fixed

### ISSUE-001: Realtime Channel Race Condition
- **File:** `src/hooks/data/shared/useFundAUM.ts`
- **Problem:** React StrictMode double-mounting caused `supabase.channel("fund-aum-realtime")` to return an already-subscribed channel singleton, triggering: `cannot add 'postgres_changes' callbacks for realtime:fund-aum-realtime after 'subscribe()'`
- **Fix:** Used `useRef` to generate a unique channel name per hook instance, and added `channel.unsubscribe()` before `removeChannel()` in cleanup.
- **Commit:** `fix(qa): ISSUE-001 — prevent realtime channel subscribe-after-subscribe race`

### ISSUE-002: Temporal Dead Zone (TDZ) Error in useTransactionSelection
- **File:** `src/features/admin/transactions/hooks/useTransactionSelection.ts`
- **Problem:** `selectableIds` was referenced in a `useEffect` dependency array and body before it was declared via `useMemo`, causing: `ReferenceError: Cannot access 'selectableIds' before initialization`
- **Fix:** Moved `selectableRows` and `selectableIds` `useMemo` declarations before the `useEffect` that depends on them.
- **Commit:** `fix(qa): ISSUE-002 — fix TDZ error in useTransactionSelection hook`

### ISSUE-003: Missing parseFinancial Import
- **File:** `src/features/admin/yields/components/DistributeYieldDialog.tsx`
- **Problem:** `parseFinancial` was used on lines 205 and 213 but never imported, causing: `ReferenceError: parseFinancial is not defined`
- **Fix:** Added `import { parseFinancial } from "@/utils/financial";`
- **Commit:** `fix(qa): ISSUE-003 — add missing parseFinancial import in DistributeYieldDialog`

### ISSUE-004: check_aum_reconciliation RPC Parameter Mismatch
- **File:** `src/features/admin/funds/services/aumReconciliationService.ts`, `src/contracts/rpcSignatures.ts`
- **Problem:** Frontend called `check_aum_reconciliation` with `p_tolerance_pct` but database function expects `p_tolerance`. Caused 404 errors and blocked yield distribution confirmation.
- **Fix:** Changed parameter name to `p_tolerance` in both call site and RPC signature contract.
- **Commit:** `fix(qa): ISSUE-004 — correct check_aum_reconciliation RPC parameter name`

### ISSUE-005: Realtime Channel Race Condition (3 More Hooks)
- **Files:**
  - `src/features/admin/shared/hooks/useAdminStats.ts`
  - `src/features/admin/system/hooks/useRealtimeAlerts.ts`
  - `src/hooks/data/shared/useRealtimeNotifications.ts`
- **Problem:** Same class of bug as ISSUE-001 — fixed channel names under React StrictMode caused singleton reuse and "cannot add callbacks after subscribe()" crashes.
- **Fix:** Applied unique channel name per hook instance (via `useRef` counter) and added `channel.unsubscribe()` before `removeChannel()` in cleanup.
- **Commit:** `fix(qa): ISSUE-005 — prevent realtime channel subscribe-after-subscribe race in 3 more hooks`

### ISSUE-006: Dead Excel Import Code
- **Files:** `src/lib/validation/excelParser.ts`, `supabase/functions/excel_import/index.ts`
- **Problem:** Unused Excel import code with hardcoded test data (investor names, yield amounts) was sitting in the codebase. Not referenced by any route or component.
- **Fix:** Deleted both files and removed `xlsx` dependency from package.json.
- **Commit:** `fix(qa): remove dead Excel import code`

### ISSUE-007: Missing reset-positions Edge Function
- **File:** `supabase/functions/reset-positions/index.ts`
- **Problem:** Frontend called `reset-positions` Edge Function but it didn't exist in `supabase/functions/`, causing 404 errors in system admin panel.
- **Fix:** Created the Edge Function with `history`, `preview`, and `execute` actions. Uses real DB counts for preview. Returns empty history if `position_resets` table doesn't exist yet.
- **Commit:** `fix(qa): add missing reset-positions Edge Function`

### ISSUE-008: DOM Nesting Warning in InvestorsTableHeader
- **File:** `src/features/admin/investors/components/list/InvestorsTableHeader.tsx`
- **Problem:** `<div>` element was nested inside `<TableHead>` (which renders as `<th>`). While technically valid HTML5, React's development mode warns about this DOM nesting.
- **Fix:** Removed the wrapper `<div>` and applied `flex items-center gap-2` classes directly to `<TableHead>` via the `className` prop.
- **Commit:** `fix(qa): ISSUE-008 — fix DOM nesting warning in InvestorsTableHeader`

## Known Issues (Not Fixed — Require Backend/Environment)

### Backend: admin-user-management Edge Function 503
- **Symptom:** `Edge Function returned a non-2xx status code` (503 Service Temporarily Unavailable)
- **Root Cause:** Local Supabase Edge Functions runtime was not running. Note: the actual function name is `admin-user-management`, not `createInvestorUser`.
- **Impact:** Cannot create new investors via the "Add Investor" wizard; report generation and email delivery also affected.
- **Action Required:** Ensure `supabase functions serve` is running. Function is present and works when the runtime is up.
